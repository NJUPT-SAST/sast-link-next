## Overview

All CI/CD runs on GitHub Actions. Workflows are defined in `.github/workflows/`.

## quality.yml

Triggered through `ci.yml` and runnable manually.

Steps:
1. Checkout code.
2. Install pnpm and Node.js.
3. Run `pnpm install`.
4. Run `pnpm lint`.
5. Run `pnpm exec tsc --noEmit`.

## test.yml

Triggered through `ci.yml` and runnable manually.

Steps:
1. Checkout code.
2. Install pnpm and Node.js.
3. Run `pnpm install`.
4. Run `pnpm test:coverage`.
5. Run `pnpm build`.
6. Upload coverage and `out/` artifacts.

## deploy.yml (optional)

Optional deployment workflow for static hosting. Disabled by default until repository secrets and target environment are configured.

## release.yml

Triggered on tags matching `v*` (for example, `v1.0.0`).

Steps:
1. Run `quality.yml`.
2. Run `test.yml`.
3. Download workflow artifacts.
4. Create a draft GitHub Release and attach build artifacts.

## Required secrets

| Secret | Used by | Description |
|---|---|---|
| `GITHUB_TOKEN` | All workflows | Automatically provided by GitHub Actions |
| Deploy credentials | `deploy.yml` | Varies by host; configure only if deploy workflow is enabled |
