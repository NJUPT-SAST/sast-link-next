## Overview

All CI/CD runs on GitHub Actions. Workflows are defined in `.github/workflows/`.

## quality.yml

Triggered on every push and pull request to `master`.

Steps:
1. Checkout code.
2. Install pnpm and Node.js.
3. Run `pnpm install`.
4. Run `pnpm lint` — fails the workflow on any ESLint error.
5. Run `pnpm exec tsc --noEmit` — fails on any TypeScript type error.

This workflow enforces baseline code quality before any merge.

## test.yml

Triggered on every push and pull request to `master`.

Steps:
1. Checkout code.
2. Install pnpm and Node.js.
3. Run `pnpm install`.
4. Run `pnpm test:coverage` — fails if coverage thresholds are not met.
5. Run `pnpm build` — verifies the static export produces a valid `out/` directory.

## build-tauri.yml

Triggered manually or on release tags (`v*`). Builds native installers for four platforms in parallel:

| Platform | Runner | Output |
|---|---|---|
| Windows | `windows-latest` | NSIS installer, MSI |
| macOS (Intel) | `macos-13` | DMG, app bundle |
| macOS (Apple Silicon) | `macos-latest` | DMG, app bundle |
| Linux | `ubuntu-22.04` | AppImage, DEB |

Each job:
1. Installs the Rust toolchain (stable) and system dependencies for the platform.
2. Installs pnpm and Node.js.
3. Runs `pnpm install`.
4. Runs `pnpm tauri build` (which internally runs `pnpm build` first via `beforeBuildCommand`).
5. Uploads the built installers as workflow artifacts.

## deploy.yml (optional)

An optional workflow for deploying the web application to a static host. Not active by default — configure the target host and credentials in repository secrets before enabling.

## release.yml

Triggered on tags matching `v*` (e.g., `v1.0.0`).

Steps:
1. Runs `build-tauri.yml` for all four platforms.
2. Downloads all installer artifacts.
3. Creates a GitHub Release with the tag name and uploads all installers as release assets.

## Required secrets

| Secret | Used by | Description |
|---|---|---|
| `GITHUB_TOKEN` | All workflows | Automatically provided by GitHub Actions |
| Deploy credentials | `deploy.yml` | Varies by target host; configure if using deploy workflow |
