---
title: Development Commands
description: All available pnpm scripts and what they do
---

All commands are run from the repository root unless noted otherwise.

## Web application

```bash
# Start the Next.js development server on port 3000 with Fast Refresh
pnpm dev

# Build the application as a static export to the out/ directory
# Used in CI and static deployment
pnpm build

# Serve the previously built static export (requires pnpm build first)
pnpm start
```

## Code quality

```bash
# Run ESLint across the entire codebase (configured in eslint.config.mjs)
pnpm lint

# Run the TypeScript compiler in type-check-only mode (no emit)
pnpm exec tsc --noEmit
```

## Testing

```bash
# Run all Jest tests once
pnpm test

# Run Jest in watch mode, re-running only changed tests on each file save
pnpm test:watch

# Run Jest and collect coverage. Fails if thresholds are not met:
#   branches: 60%  functions: 60%  lines: 70%  statements: 70%
pnpm test:coverage
```

## shadcn/ui components

```bash
# Add a new shadcn/ui-style primitive to components/ui/
pnpm dlx shadcn@latest add <component-name>
```

## Documentation site

The documentation site lives in the `docs/` subdirectory and has its own `package.json`.

```bash
cd docs

# Start the docs development server (default port: 3000, or set a different one)
pnpm dev

# Build the documentation site for production
pnpm build
```

## Build output

`pnpm build` (main app) produces a fully static site in `out/`.
