## Overview

Password reset follows the same four-step structure as registration. The main differences are:

- The account is looked up by existing username or email (not a new student ID).
- A different ticket type (`resetPwdTicket`) is issued.
- A different header (`RESETPWD-TICKET`) is used in subsequent requests.
- The final API call is `resetPassword` rather than `userRegister`.

```
Step 1: Enter account  →  Step 2: Verify email  →  Step 3: Set new password  →  Step 4: Success
```

## Step 1 — Account lookup

The user enters their username or email. On submission:

1. `verifyResetAccount(username)` calls `GET /verify/account?username=...&flag=2`.
2. A `resetPwdTicket` is returned on success.
3. `sendMail(ticket, "reset")` sends the verification email using `RESETPWD-TICKET` header.
4. The ticket is stored in component state and the flow advances to step 2.

The `loginAccountSchema` validator (9-digit student ID or email) is reused here.

## Step 2 — Email verification

Identical behavior to registration step 2, except:
- The `RESETPWD-TICKET` header is used instead of `REGISTER-TICKET`.
- `verifyCaptcha(ticket, captcha, "reset")` is called with the `"reset"` type argument.

The 60-second resend cooldown and `"S-"` captcha prefix are the same.

## Step 3 — New password

The user sets a new password subject to the same rules as registration:

- Minimum 8 characters.
- Must contain at least one letter and at least one number.
- Confirmation field must match.

On submission, `resetPassword(password, resetTicket)` posts to `POST /user/resetPassword`:
- Body field: `newPassword` (differs from registration which uses `password`).
- Header: `RESETPWD-TICKET`.

## Step 4 — Success screen

The same animated success screen as registration. Includes a link to `/login`.

## API difference summary

| Aspect | Registration | Password Reset |
|---|---|---|
| Verify endpoint flag | `flag=0` | `flag=2` |
| Ticket field name | `registerTicket` | `resetPwdTicket` |
| Header name | `REGISTER-TICKET` | `RESETPWD-TICKET` |
| Final endpoint | `POST /user/register` | `POST /user/resetPassword` |
| Final body field | `password` | `newPassword` |
