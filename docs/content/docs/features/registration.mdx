## Overview

New accounts are created through a four-step registration flow. Each step advances only after the current step's validation and API call succeed. Steps are tracked with a local `step` state in `app/(tourist)/register/page.tsx`.

```
Step 1: Enter student ID  →  Step 2: Verify email  →  Step 3: Set password  →  Step 4: Success
```

## Step 1 — Student ID input

The user enters their 9-digit student ID. On submission, the frontend:

1. Derives the institutional email address: `studentId@njupt.edu.cn` (via `studentIdToEmail(studentId)` in `lib/validations/auth.ts`).
2. Calls `verifyRegistAccount(username)`, hitting `GET /verify/account?username=...&flag=0`.
3. Receives a `registerTicket` on success.
4. Calls `sendMail(registerTicket)` to send the verification email.
5. Stores the ticket and advances to step 2.

```ts
// lib/validations/auth.ts
export const registerStudentIdSchema = z.string().regex(/^\d{9}$/, "Must be exactly 9 digits");

export function studentIdToEmail(studentId: string): string {
  return `${studentId}@njupt.edu.cn`;
}
```

The derived email is displayed to the user so they know where to expect the verification code.

## Step 2 — Email verification

The user enters the 5-digit code received by email. The frontend prepends the fixed `"S-"` prefix before sending:

```ts
// lib/api/auth.ts
params.append("captcha", "S-" + captcha);
```

The call is `POST /verify/captcha` with `REGISTER-TICKET` header and the `application/x-www-form-urlencoded` body.

A resend button is available with a 60-second cooldown enforced by a countdown timer in the `VerificationCodeInput` component. When the timer reaches zero, the button re-enables and calls `sendMail(ticket)` again.

The `verificationCodeSchema` validates that the input is exactly 5 digits before submission.

## Step 3 — Password setup

The user sets their password. Validation rules:

- Minimum 8 characters.
- Must contain at least one letter and at least one number.
- A confirmation field must match the password exactly.

```ts
// lib/validations/auth.ts
export const passwordSchema = z
  .string()
  .min(8, "At least 8 characters")
  .regex(/[a-zA-Z]/, "Must contain a letter")
  .regex(/[0-9]/, "Must contain a number");

export const registerPasswordSchema = z
  .object({ password: passwordSchema, confirmPassword: z.string() })
  .refine((d) => d.password === d.confirmPassword, {
    message: "Passwords do not match",
    path: ["confirmPassword"],
  });
```

On submission, `userRegister(password, registerTicket)` posts to `POST /user/register` with the password in the body and `REGISTER-TICKET` in the header. Success advances to step 4.

## Step 4 — Success screen

Step 4 is a confirmation screen showing an animated checkmark. It includes a link to `/login` so the user can immediately sign in with their new account. No API call is made in this step.

## Error handling

Each step displays inline error messages from:
- Zod validation (field-level errors via React Hook Form).
- API error responses (`ResType.ErrMsg` from the backend).

If an API call fails, the step does not advance and the user can retry.
