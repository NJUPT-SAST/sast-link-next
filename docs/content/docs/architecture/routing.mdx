## Route groups

Next.js route groups (folders wrapped in parentheses) organize pages without affecting the URL path. sast-link-next uses two groups:

### (tourist)

All unauthenticated pages live under `app/(tourist)/`. These share a single background layout (`app/(tourist)/layout.tsx`) that applies a gradient background.

| URL | Component | Description |
|---|---|---|
| `/` | `app/page.tsx` | Account switcher and quick-login entry |
| `/login` | `app/(tourist)/login/page.tsx` | Two-step login flow |
| `/register` | `app/(tourist)/register/page.tsx` | Four-step registration flow |
| `/reset` | `app/(tourist)/reset/page.tsx` | Four-step password reset flow |
| `/callback/feishu` | `app/(tourist)/callback/feishu/page.tsx` | Feishu OAuth callback |
| `/callback/github` | `app/(tourist)/callback/github/page.tsx` | GitHub OAuth callback |

### (user)

Authenticated pages live under `app/(user)/`. The layout at `app/(user)/home/layout.tsx` calls `useFetchProfile()` on mount. If no token exists in localStorage, the user is redirected to `/login`.

| URL | Component | Description |
|---|---|---|
| `/home` | `app/(user)/home/page.tsx` | Profile overview, quick actions, apps |
| `/home/edit` | `app/(user)/home/edit/page.tsx` | Avatar upload and profile editing |
| `/home/edit/safety` | `app/(user)/home/edit/safety/page.tsx` | Safety settings |

## Parallel routes

The `/home` layout uses [Next.js parallel routes](https://nextjs.org/docs/app/building-your-application/routing/parallel-routes) to compose the side panels independently of the main content:

```
app/(user)/home/
├── layout.tsx          ← defines @infoPanel and @profilePanel slots
├── page.tsx            ← main content
├── @infoPanel/
│   └── default.tsx     ← rendered in the infoPanel slot
└── @profilePanel/
    └── default.tsx     ← rendered in the profilePanel slot
```

The layout receives `infoPanel` and `profilePanel` as props and arranges them in a responsive grid alongside the main `children`. Each slot renders independently; navigation within `/home` does not unmount the panels.

## Layout hierarchy

```
app/layout.tsx              (root: font, Providers)
  └── app/(tourist)/layout.tsx     (background gradient)
        └── app/(tourist)/login/page.tsx
        └── app/(tourist)/register/page.tsx
        └── ...

  └── app/(user)/home/layout.tsx   (TopBar + parallel panels, useFetchProfile)
        └── app/(user)/home/page.tsx
        └── app/(user)/home/edit/layout.tsx
              └── app/(user)/home/edit/page.tsx
              └── app/(user)/home/edit/safety/page.tsx
```

## File naming conventions

- Pages: `page.tsx` in each route folder.
- Layouts: `layout.tsx` in each route folder.
- Route-private components: `_components/` subdirectory inside the route folder. The underscore prefix prevents Next.js from treating the folder as a route segment.
- Parallel route slots: folders prefixed with `@` (e.g., `@infoPanel/`). Each must contain at least a `default.tsx` to avoid an unmatched slot error during navigation.
