## Tailwind CSS v4

sast-link-next uses Tailwind CSS v4, which is configured through PostCSS (`postcss.config.mjs`) rather than a `tailwind.config.js` file:

```js
// postcss.config.mjs
export default {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
```

Tailwind v4 reads the design tokens and theme from `app/globals.css` using the `@theme` directive instead of a JavaScript configuration object.

## CSS variables

`app/globals.css` defines all design tokens as CSS custom properties under the `:root` and `.dark` selectors. These variables control colors, border radii, and other primitives used throughout the application. The naming follows the shadcn/ui convention (e.g., `--background`, `--foreground`, `--primary`, `--muted`).

Components reference these variables through Tailwind utility classes mapped to the CSS variables (e.g., `bg-background`, `text-foreground`, `text-muted-foreground`).

## Animation utilities

`tw-animate-css` is imported in `app/globals.css` to provide animation utility classes (`animate-in`, `animate-out`, `fade-in`, `fade-out`, `slide-in-from-*`, etc.). These are used by dialog, sheet, and dropdown components for enter/exit transitions.

Framer Motion (`framer-motion`) is used in `components/animation/page-transition.tsx` for page-level transitions between routes.

## Component patterns (shadcn/ui-style)

`components/ui/` contains primitive UI components following the shadcn/ui pattern:

- Each component is a thin TypeScript wrapper around a Radix UI primitive (or a custom implementation for simple components).
- Components accept a `className` prop and use `cn()` to merge it with the component's default classes.
- No runtime theme configuration is needed; all styling comes from Tailwind utilities and CSS variables.

Components are added with:

```bash
pnpm dlx shadcn@latest add <component-name>
```

This generates a file in `components/ui/` that can be edited freely.

## The cn() utility

`lib/utils.ts` exports `cn`, a function that combines `clsx` and `tailwind-merge`:

```ts
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

Usage in components:

```tsx
<div className={cn("base-classes", condition && "conditional-class", className)}>
```

`tailwind-merge` resolves Tailwind class conflicts (e.g., `px-2 px-4` becomes `px-4`) so that callers can reliably override defaults by passing a `className` prop.

## Dark mode

`next-themes` provides the `ThemeProvider` in `app/providers.tsx`. It injects a `class` or `data-theme` attribute on the `<html>` element, which Tailwind uses to apply `.dark` selector styles.

The `ThemeToggle` component in `components/layout/theme-toggle.tsx` exposes the toggle control.
