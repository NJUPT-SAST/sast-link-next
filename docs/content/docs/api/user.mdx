---
title: User API
description: Functions in lib/api/user.ts for login, logout, profile management, and avatar upload
---

All functions use the shared `apiClient` from `lib/api/client.ts`. Authenticated endpoints rely on the `Token` header injected by the Axios request interceptor.

## userLogin

```ts
function userLogin(
  password: string,
  loginTicket: string,
  oauthTicket?: string | null,
): Promise<AxiosResponse<ResType<{ loginToken: string }>>>
```

Authenticates the user and returns a session token.

- **Method**: `POST /user/login`
- **Body**: `multipart/form-data` with field `password`
- **Header**: `LOGIN-TICKET: <loginTicket>`
- **Header (optional)**: `OAUTH-TICKET: <oauthTicket>` — present only when the user is completing an OAuth account binding flow
- **Returns**: `loginToken` — persist to localStorage with `setToken(token)`.

## getUserInfo

```ts
function getUserInfo(): Promise<AxiosResponse<ResType<{ email: string; userId: string }>>>
```

Returns the basic identity of the authenticated user.

- **Method**: `GET /user/info`
- **Auth**: requires `Token` header
- **Returns**: `email` and `userId`.

## userLogout

```ts
function userLogout(): Promise<AxiosResponse<ResType<null>>>
```

Invalidates the current session on the backend.

- **Method**: `POST /user/logout`
- **Auth**: requires `Token` header
- **Returns**: `null` on success.

After calling this, remove the token from localStorage via `clearToken()` and reset `useAuthStore` via `logout()`.

## getUserProfile

```ts
function getUserProfile(): Promise<AxiosResponse<ResType<UserProfileType>>>
```

Returns the full profile of the authenticated user.

- **Method**: `GET /profile/getProfile`
- **Auth**: requires `Token` header
- **Returns**: `UserProfileType` — see type definition in [API Client](/docs/api).

This is the only profile fetch in the application. It is called by `useFetchProfile()` in `hooks/use-fetch-profile.ts`, which writes the result into all three relevant Zustand stores.

## editProfile

```ts
function editProfile(data: EditableProfileType): Promise<AxiosResponse<ResType<null>>>
```

Updates the authenticated user's profile fields.

- **Method**: `POST /profile/changeProfile`
- **Auth**: requires `Token` header
- **Body**: JSON with any subset of `EditableProfileType` fields
- **Returns**: `null` on success.

After a successful call, update the local store with `useUserProfileStore.updateProfile(fields)` to keep the UI in sync.

## uploadAvatar

```ts
function uploadAvatar(file: Blob): Promise<AxiosResponse<ResType<{ filePath: string }>>>
```

Uploads a new avatar image.

- **Method**: `POST /profile/uploadAvatar`
- **Auth**: requires `Token` header
- **Body**: `multipart/form-data` with field `avatarFile`
- **Returns**: `filePath` — the URL of the stored avatar image.

The `Blob` is typically produced by the `react-avatar-editor` canvas export in `app/(user)/home/edit/page.tsx`.

## getUserBindStatus

```ts
function getUserBindStatus(): Promise<AxiosResponse<ResType<string[]>>>
```

Returns a list of OAuth provider identifiers currently linked to the authenticated user's account.

- **Method**: `GET /profile/bindStatus`
- **Auth**: requires `Token` header
- **Returns**: Array of provider name strings (e.g., `["github", "feishu"]`).

Used by the `BindAppItem` component to display linked accounts in the profile UI.
