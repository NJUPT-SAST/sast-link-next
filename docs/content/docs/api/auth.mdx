---
title: Auth API
description: Functions in lib/api/auth.ts for account verification, email captcha, registration, and password reset
---

All functions use the shared `apiClient` from `lib/api/client.ts`.

## verifyRegistAccount

```ts
function verifyRegistAccount(username: string): Promise<AxiosResponse<ResType<{ registerTicket: string }>>>
```

Checks whether the given username is available for registration.

- **Method**: `GET /verify/account`
- **Query**: `username`, `flag=0`
- **Returns**: `registerTicket` — used in all subsequent registration requests.

## verifyLoginAccount

```ts
function verifyLoginAccount(username: string): Promise<AxiosResponse<ResType<{ loginTicket: string }>>>
```

Checks whether the given username exists for login.

- **Method**: `GET /verify/account`
- **Query**: `username`, `flag=1`
- **Returns**: `loginTicket` — passed to `userLogin` in the next step.

## verifyResetAccount

```ts
function verifyResetAccount(username: string): Promise<AxiosResponse<ResType<{ resetPwdTicket: string }>>>
```

Checks whether the given username exists for password reset.

- **Method**: `GET /verify/account`
- **Query**: `username`, `flag=2`
- **Returns**: `resetPwdTicket` — used in all subsequent reset requests.

## sendMail

```ts
function sendMail(ticket: string, type?: "reset"): Promise<AxiosResponse<ResType<null>>>
```

Sends a verification email to the address associated with the ticket.

- **Method**: `GET /sendEmail`
- **Header (registration)**: `REGISTER-TICKET: <ticket>`
- **Header (reset)**: `RESETPWD-TICKET: <ticket>` (when `type === "reset"`)
- **Returns**: `null` on success.

Called after `verifyRegistAccount` or `verifyResetAccount`, and again when the user clicks "Resend" on the verification step.

## verifyCaptcha

```ts
function verifyCaptcha(
  ticket: string,
  captcha: string,
  type?: "reset",
): Promise<AxiosResponse<ResType<null>>>
```

Verifies the 5-digit code entered by the user. The `"S-"` prefix is prepended internally before sending.

- **Method**: `POST /verify/captcha`
- **Content-Type**: `application/x-www-form-urlencoded;charset=UTF-8`
- **Body**: `captcha=S-<code>`
- **Header (registration)**: `REGISTER-TICKET: <ticket>`
- **Header (reset)**: `RESETPWD-TICKET: <ticket>`
- **Returns**: `null` on success.

## userRegister

```ts
function userRegister(password: string, registerTicket: string): Promise<AxiosResponse<ResType<null>>>
```

Creates a new user account.

- **Method**: `POST /user/register`
- **Content-Type**: `application/x-www-form-urlencoded`
- **Body**: `password=<password>`
- **Header**: `REGISTER-TICKET: <registerTicket>`
- **Returns**: `null` on success.

Called after `verifyCaptcha` succeeds during the registration flow.

## resetPassword

```ts
function resetPassword(password: string, resetTicket: string): Promise<AxiosResponse<ResType<null>>>
```

Sets a new password for an existing account.

- **Method**: `POST /user/resetPassword`
- **Content-Type**: `application/x-www-form-urlencoded`
- **Body**: `newPassword=<password>` (note: field name differs from `userRegister`)
- **Header**: `RESETPWD-TICKET: <resetTicket>`
- **Returns**: `null` on success.

## getFeishuLoginStatus

```ts
function getFeishuLoginStatus(code: string, state: string): Promise<AxiosResponse<any>>
```

Exchanges the Feishu OAuth authorization code for a session token or OAuth ticket.

- **Method**: `GET /login/lark/callback`
- **Query**: `code`, `state`
- **Returns**: `{ loginToken }` (direct login) or `{ oauthTicket }` (account binding required).

Called from `app/(tourist)/callback/feishu/page.tsx`.
