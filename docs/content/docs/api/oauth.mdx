## Overview

`lib/api/oauth.ts` provides a single function, `oAuth()`, used when a third-party application requests authorization through the SAST Link OAuth server. This is distinct from the social login flow (GitHub/Feishu), which is handled by the callback routes and `lib/api/auth.ts`.

## oAuth

```ts
interface OAuthParams {
  client_id: string | null;
  code_challenge: string | null;
  code_challenge_method: string | null;
  redirect_uri: string | null;
  response_type: string | null;
  scope: string | null;
  state: string | null;
}

function oAuth(data: OAuthParams): void
```

Constructs a PKCE authorization URL and redirects the browser to it via `window.location.href`.

- **Target URL**: `${NEXT_PUBLIC_API_BASE_URL}/oauth2/authorize`
- **Parameters**: All non-null fields from `OAuthParams` are appended as query parameters.
- **`part` parameter**: If a session token is present in localStorage, it is appended as `part=<token>` so the authorization server can identify the approving user without requiring a separate login step.

### Usage

This function is typically invoked when the application receives an OAuth authorization request as the identity provider. The caller constructs `OAuthParams` from the incoming request's query string and passes them to `oAuth()`.

```ts
oAuth({
  client_id: searchParams.get("client_id"),
  code_challenge: searchParams.get("code_challenge"),
  code_challenge_method: searchParams.get("code_challenge_method"),
  redirect_uri: searchParams.get("redirect_uri"),
  response_type: searchParams.get("response_type"),
  scope: searchParams.get("scope"),
  state: searchParams.get("state"),
});
```

Null values are omitted from the URL; the function iterates all entries and skips falsy values.

### Side effects

The function calls `window.location.href = ...`, which causes a full browser navigation. There is no return value and no Promise. After the user approves or denies on the authorization server, the server redirects the browser back to `redirect_uri` with the authorization code.

## Related

- Social login callbacks (GitHub, Feishu) use `getGithubLoginStatus` and `getFeishuLoginStatus` in `lib/api/auth.ts`, not this function.
- The `NEXT_PUBLIC_API_BASE_URL` environment variable must be set correctly for the redirect target to resolve.
