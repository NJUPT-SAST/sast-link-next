## Overview

All backend API communication goes through a single Axios instance defined in `lib/api/client.ts`. Higher-level functions in `lib/api/auth.ts`, `lib/api/user.ts`, and `lib/api/oauth.ts` wrap this client.

## Axios instance

```ts
// lib/api/client.ts
import axios from "axios";
import { getToken } from "@/lib/token";

const API_BASE_URL = process.env.NEXT_PUBLIC_API_BASE_URL ?? "";

export const apiClient = axios.create({
  baseURL: API_BASE_URL,
});

apiClient.interceptors.request.use((config) => {
  const token = getToken();
  if (token) {
    config.headers.Token = token;
  }
  return config;
});
```

- `baseURL` is set from `NEXT_PUBLIC_API_BASE_URL`. If the variable is not set, the base URL is an empty string and all paths are relative to the current origin.
- The request interceptor reads the current token from localStorage on every request. This means token updates take effect immediately without reconfiguring the client.
- The header name is `Token` (capital T), not the standard `Authorization: Bearer` pattern.

## Token storage

Token helpers live in `lib/token.ts`:

```ts
getToken(): string | null   // reads from localStorage
setToken(token: string): void
clearToken(): void
```

## Response type

Every backend endpoint follows the same response envelope:

```ts
// lib/api/types.ts
type ResType<T> = ResponseErrorType | ResponseSuccessType<T>;

interface ResponseSuccessType<T> {
  Success: true;
  Data: T;
}

interface ResponseErrorType {
  Success: false;
  ErrCode: number;
  ErrMsg: string;
  Data: null;
}
```

All API functions return `Promise<AxiosResponse<ResType<T>>>`. Callers check `res.data.Success` before accessing `res.data.Data`:

```ts
const res = await getUserProfile();
if (res.data.Success) {
  const profile = res.data.Data; // typed as UserProfileType
} else {
  console.error(res.data.ErrMsg);
}
```

## Shared types

```ts
// lib/api/types.ts

interface UserProfileType {
  nickname: string;
  dep: string | null;
  org: string | null;
  email: string;
  avatar: string | null;
  bio: string | null;
  link: string[] | null;
  badge: { title: string; description: string; create_at: string }[] | null;
  hide: string[] | null;
}

interface EditableProfileType {
  nickname?: string | null;
  org_id?: number;
  bio?: string;
  link?: string[] | null;
  hide?: string[] | null;
}

interface UserAccount {
  token: string;
  avatar?: string | null;
  nickName?: string;
  email: string;
  userId: string;
}

interface CurrentUserProfile {
  username: string;
  email: string;
  userId?: string;
  avatar?: string;
  organization?: string;
  description?: string;
  link?: string;
}
```
