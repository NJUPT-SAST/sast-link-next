## Stack

| Tool | Version | Purpose |
|---|---|---|
| Jest | 30 | Test runner and assertion library |
| jest-fixed-jsdom | latest | DOM environment with fixes for async/structuredClone |
| ts-jest | latest | TypeScript transformation for test files |
| @testing-library/react | latest | React component rendering and interaction |
| @testing-library/user-event | latest | Simulates user interactions |
| MSW | 2 | Network-layer API mocking |

## Configuration

### jest.config.ts

```ts
// Key settings
testEnvironment: "jest-fixed-jsdom"
transform: { "^.+\\.(t|j)sx?$": ["ts-jest", { tsconfig: "tsconfig.json" }] }
moduleNameMapper: { "^@/(.*)$": "<rootDir>/$1" }  // resolves @/* imports
```

The `jest-fixed-jsdom` environment is used instead of the standard `jsdom` because it includes patches for `structuredClone` and other globals that React 19 and some testing patterns require.

### jest.setup.ts

The setup file runs before each test suite and:

1. Imports `@testing-library/jest-dom` to add custom matchers (`toBeInTheDocument`, `toHaveClass`, etc.).
2. Mocks `next/image` with a simple `<img>` element.
3. Mocks `next/navigation` (`useRouter`, `usePathname`, `useSearchParams`).
4. Starts the MSW server before all tests, resets handlers between tests, and closes the server after all tests.

### MSW server lifecycle

```ts
// jest.setup.ts
beforeAll(() => server.listen())
afterEach(() => server.resetHandlers())
afterAll(() => server.close())
```

`server.resetHandlers()` removes any test-specific handler overrides added via `server.use(...)`, ensuring handler state does not leak between test files.

## Coverage thresholds

`pnpm test:coverage` fails if any threshold is not met:

| Metric | Threshold |
|---|---|
| Branches | 60% |
| Functions | 60% |
| Lines | 70% |
| Statements | 70% |

Coverage is collected from all `*.ts` and `*.tsx` files in `app/`, `components/`, `lib/`, `store/`, and `hooks/`, excluding test files, type definition files, and generated code.

## Test file colocation

Test files are colocated with the source files they test, using the `.test.ts` or `.test.tsx` extension:

```
components/auth/auth-shell.tsx
components/auth/auth-shell.test.tsx

lib/api/auth.ts
lib/api/auth.test.ts

store/use-auth-store.ts
store/use-auth-store.test.ts
```

Higher-level integration tests for pages live alongside the page file:

```
app/(tourist)/login/page.tsx
app/(tourist)/login/page.test.tsx
```

## Running tests

```bash
# Run all tests once
pnpm test

# Watch mode: reruns only affected tests on file save
pnpm test:watch

# Run with coverage report
pnpm test:coverage
```
