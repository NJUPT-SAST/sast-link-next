## 概述

所有后端 API 通信均通过 `lib/api/client.ts` 中定义的单一 Axios 实例进行。`lib/api/auth.ts`、`lib/api/user.ts` 和 `lib/api/oauth.ts` 中的高层函数封装此客户端。

## Axios 实例

```ts
// lib/api/client.ts
import axios from "axios";
import { getToken } from "@/lib/token";

const API_BASE_URL = process.env.NEXT_PUBLIC_API_BASE_URL ?? "";

export const apiClient = axios.create({
  baseURL: API_BASE_URL,
});

apiClient.interceptors.request.use((config) => {
  const token = getToken();
  if (token) {
    config.headers.Token = token;
  }
  return config;
});
```

- `baseURL` 从 `NEXT_PUBLIC_API_BASE_URL` 读取。若未设置，则所有路径相对于当前源。
- 请求拦截器在每次请求时从 localStorage 读取当前令牌，令牌更新后无需重新配置客户端即可立即生效。
- 请求头名称为 `Token`（首字母大写），而非标准的 `Authorization: Bearer` 格式。

## Token 存储

Token 辅助函数位于 `lib/token.ts`：

```ts
getToken(): string | null   // 从 localStorage 读取
setToken(token: string): void
clearToken(): void
```

## 响应类型

每个后端端点均遵循相同的响应信封格式：

```ts
// lib/api/types.ts
type ResType<T> = ResponseErrorType | ResponseSuccessType<T>;

interface ResponseSuccessType<T> {
  Success: true;
  Data: T;
}

interface ResponseErrorType {
  Success: false;
  ErrCode: number;
  ErrMsg: string;
  Data: null;
}
```

所有 API 函数返回 `Promise<AxiosResponse<ResType<T>>>`。调用方在访问 `res.data.Data` 前须检查 `res.data.Success`：

```ts
const res = await getUserProfile();
if (res.data.Success) {
  const profile = res.data.Data; // 类型为 UserProfileType
} else {
  console.error(res.data.ErrMsg);
}
```

## 共享类型

```ts
// lib/api/types.ts

interface UserProfileType {
  nickname: string;
  dep: string | null;
  org: string | null;
  email: string;
  avatar: string | null;
  bio: string | null;
  link: string[] | null;
  badge: { title: string; description: string; create_at: string }[] | null;
  hide: string[] | null;
}

interface EditableProfileType {
  nickname?: string | null;
  org_id?: number;
  bio?: string;
  link?: string[] | null;
  hide?: string[] | null;
}

interface UserAccount {
  token: string;
  avatar?: string | null;
  nickName?: string;
  email: string;
  userId: string;
}

interface CurrentUserProfile {
  username: string;
  email: string;
  userId?: string;
  avatar?: string;
  organization?: string;
  description?: string;
  link?: string;
}
```
