## 概述

`lib/api/oauth.ts` 提供单一函数 `oAuth()`，用于第三方应用请求通过 SAST Link OAuth 服务器进行授权的场景。这与社交登录流程（GitHub/飞书）不同，后者由回调路由和 `lib/api/auth.ts` 处理。

## oAuth

```ts
interface OAuthParams {
  client_id: string | null;
  code_challenge: string | null;
  code_challenge_method: string | null;
  redirect_uri: string | null;
  response_type: string | null;
  scope: string | null;
  state: string | null;
}

function oAuth(data: OAuthParams): void
```

构建 PKCE 授权 URL 并通过 `window.location.href` 跳转浏览器至目标地址。

- **目标 URL**：`${NEXT_PUBLIC_API_BASE_URL}/oauth2/authorize`
- **参数**：`OAuthParams` 中所有非 null 字段均作为查询参数附加。
- **`part` 参数**：若 localStorage 中存在会话令牌，则将其作为 `part=<token>` 附加，使授权服务器能在无需额外登录的情况下识别当前审批用户。

### 使用示例

通常在应用以身份提供方角色接收 OAuth 授权请求时调用。调用方从传入请求的查询字符串构建 `OAuthParams` 并传入 `oAuth()`：

```ts
oAuth({
  client_id: searchParams.get("client_id"),
  code_challenge: searchParams.get("code_challenge"),
  code_challenge_method: searchParams.get("code_challenge_method"),
  redirect_uri: searchParams.get("redirect_uri"),
  response_type: searchParams.get("response_type"),
  scope: searchParams.get("scope"),
  state: searchParams.get("state"),
});
```

Null 值不会附加至 URL，函数遍历所有条目并跳过假值。

### 副作用

该函数调用 `window.location.href = ...`，触发完整的浏览器导航，无返回值，无 Promise。用户在授权服务器审批或拒绝后，服务器会携带授权码将浏览器重定向回 `redirect_uri`。

## 相关内容

- 社交登录回调（GitHub、飞书）使用 `lib/api/auth.ts` 中的 `getGithubLoginStatus` 和 `getFeishuLoginStatus`，而非本函数。
- `NEXT_PUBLIC_API_BASE_URL` 环境变量须正确配置，确保重定向目标可解析。
