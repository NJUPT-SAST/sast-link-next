## 路由组

Next.js 路由组（括号包裹的文件夹）在不影响 URL 路径的前提下组织页面。sast-link-next 使用两个路由组：

### (tourist)

所有未登录页面位于 `app/(tourist)/` 下，共享一个背景布局（`app/(tourist)/layout.tsx`），统一应用渐变背景。

| URL | 组件 | 说明 |
|---|---|---|
| `/` | `app/page.tsx` | 账号切换器与快速登录入口 |
| `/login` | `app/(tourist)/login/page.tsx` | 两步登录流程 |
| `/register` | `app/(tourist)/register/page.tsx` | 四步注册流程 |
| `/reset` | `app/(tourist)/reset/page.tsx` | 四步密码重置流程 |
| `/callback/feishu` | `app/(tourist)/callback/feishu/page.tsx` | 飞书 OAuth 回调 |
| `/callback/github` | `app/(tourist)/callback/github/page.tsx` | GitHub OAuth 回调 |

### (user)

已登录页面位于 `app/(user)/` 下。`app/(user)/home/layout.tsx` 在挂载时调用 `useFetchProfile()`。若 localStorage 中无 Token，用户将被重定向至 `/login`。

| URL | 组件 | 说明 |
|---|---|---|
| `/home` | `app/(user)/home/page.tsx` | 资料概览、快捷操作、应用列表 |
| `/home/edit` | `app/(user)/home/edit/page.tsx` | 头像上传与资料编辑 |
| `/home/edit/safety` | `app/(user)/home/edit/safety/page.tsx` | 安全设置 |

## 并行路由

`/home` 布局使用 [Next.js 并行路由](https://nextjs.org/docs/app/building-your-application/routing/parallel-routes)，将侧边面板与主内容独立组合：

```
app/(user)/home/
├── layout.tsx          ← 定义 @infoPanel 和 @profilePanel 插槽
├── page.tsx            ← 主内容
├── @infoPanel/
│   └── default.tsx     ← 渲染至 infoPanel 插槽
└── @profilePanel/
    └── default.tsx     ← 渲染至 profilePanel 插槽
```

布局将 `infoPanel` 和 `profilePanel` 作为 Props 接收，在响应式网格中与主 `children` 并排排列。每个插槽独立渲染，在 `/home` 内导航不会卸载面板。

## 布局层级

```
app/layout.tsx                   (根：字体、Providers)
  └── app/(tourist)/layout.tsx   (渐变背景)
        └── app/(tourist)/login/page.tsx
        └── app/(tourist)/register/page.tsx
        └── ...

  └── app/(user)/home/layout.tsx (TopBar + 并行面板，调用 useFetchProfile)
        └── app/(user)/home/page.tsx
        └── app/(user)/home/edit/layout.tsx
              └── app/(user)/home/edit/page.tsx
              └── app/(user)/home/edit/safety/page.tsx
```

## 文件命名规范

- 页面：每个路由文件夹下的 `page.tsx`。
- 布局：每个路由文件夹下的 `layout.tsx`。
- 路由私有组件：放置在路由文件夹内的 `_components/` 子目录。下划线前缀可防止 Next.js 将该文件夹识别为路由段。
- 并行路由插槽：以 `@` 为前缀的文件夹（如 `@infoPanel/`），每个插槽至少需要包含一个 `default.tsx`，以避免导航时出现插槽未匹配错误。
