## Tailwind CSS v4

sast-link-next 使用 Tailwind CSS v4，通过 PostCSS（`postcss.config.mjs`）进行配置，无需 `tailwind.config.js` 文件：

```js
// postcss.config.mjs
export default {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
```

Tailwind v4 使用 `@theme` 指令从 `app/globals.css` 中读取设计令牌与主题配置，而非 JavaScript 配置对象。

## CSS 变量

`app/globals.css` 在 `:root` 和 `.dark` 选择器下将所有设计令牌定义为 CSS 自定义属性。这些变量控制整个应用的颜色、圆角等基础属性，命名遵循 shadcn/ui 规范（如 `--background`、`--foreground`、`--primary`、`--muted`）。

组件通过映射至 CSS 变量的 Tailwind 工具类引用这些变量（如 `bg-background`、`text-foreground`、`text-muted-foreground`）。

## 动画工具

`tw-animate-css` 在 `app/globals.css` 中引入，提供动画工具类（`animate-in`、`animate-out`、`fade-in`、`fade-out`、`slide-in-from-*` 等），被 dialog、sheet 和 dropdown 组件用于进入/退出过渡动画。

Framer Motion（`framer-motion`）在 `components/animation/page-transition.tsx` 中用于路由间的页面级过渡。

## 组件模式（shadcn/ui 风格）

`components/ui/` 包含遵循 shadcn/ui 模式的基础 UI 组件：

- 每个组件都是对 Radix UI 原语的轻量 TypeScript 封装（简单组件则使用自定义实现）。
- 组件接受 `className` prop，通过 `cn()` 将其与组件默认类合并。
- 无需运行时主题配置，所有样式均来自 Tailwind 工具类与 CSS 变量。

添加新组件：

```bash
pnpm dlx shadcn@latest add <component-name>
```

这会在 `components/ui/` 生成一个可自由编辑的文件。

## cn() 工具函数

`lib/utils.ts` 导出 `cn`，它结合了 `clsx` 和 `tailwind-merge`：

```ts
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

组件中的使用示例：

```tsx
<div className={cn("基础类", condition && "条件类", className)}>
```

`tailwind-merge` 会解决 Tailwind 类冲突（如 `px-2 px-4` 会变为 `px-4`），使调用方能够通过传入 `className` prop 可靠地覆盖默认值。

## 深色模式

`next-themes` 在 `app/providers.tsx` 中提供 `ThemeProvider`。它会在 `<html>` 元素上注入 `class` 或 `data-theme` 属性，Tailwind 据此应用 `.dark` 选择器样式。

`components/layout/theme-toggle.tsx` 中的 `ThemeToggle` 组件提供切换控制。
