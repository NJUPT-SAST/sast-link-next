## MSW 处理器模式

`mocks/handlers/` 中的模拟处理器维护内存状态，模拟真实后端的会话模型。理解此状态对编写集成测试至关重要。

### 认证处理器状态

`mocks/handlers/auth.ts` 追踪：
- 从票据字符串到 `{ username, verified, type }` 的 `tickets` 映射。
- 每个票据上的验证码验证标志。

当测试调用 `verifyRegistAccount("B21041234")` 时，处理器签发票据并存储。后续携带该票据的 `sendMail`、`verifyCaptcha` 和 `userRegister` 调用都会对照此内存映射进行校验。

### 用户处理器状态

`mocks/handlers/user.ts` 追踪：
- 预置测试账号的 `users` 映射。
- 从 `loginToken` 到 `userId` 的 `sessions` 映射。

每个已认证请求的 `Token` 请求头都会在 `sessions` 中查询，若令牌不存在或无效，处理器返回错误响应。

### 在测试中覆盖处理器

使用 `server.use(...)` 为特定测试添加一次性处理器覆盖：

```ts
import { server } from "@/mocks/server";
import { http, HttpResponse } from "msw";

test("登录失败时显示错误信息", async () => {
  server.use(
    http.post("/user/login", () =>
      HttpResponse.json({ Success: false, ErrCode: 401, ErrMsg: "凭据无效", Data: null })
    )
  );

  // ... 渲染并交互
});
```

`jest.setup.ts` 中 `afterEach` 的 `server.resetHandlers()` 会在测试完成后移除此覆盖。

## 渲染组件

使用 `@testing-library/react` 中的 `render`。使用了 Zustand Store 或 SWR 的组件无需自定义 Provider 包装——两者在测试中均可无需上下文直接工作。

```tsx
import { render, screen } from "@testing-library/react";
import { AuthShell } from "@/components/auth/auth-shell";

test("渲染标题", () => {
  render(<AuthShell heading="登录" description="请输入您的凭据" />);
  expect(screen.getByText("登录")).toBeInTheDocument();
});
```

### 使用 Next.js 路由的组件

`jest.setup.ts` 全局模拟了 `next/navigation`，`usePathname` 返回空字符串，`useRouter` 返回无操作对象。若测试需要特定路由行为，可在本地模拟该模块：

```ts
import * as navigation from "next/navigation";
jest.spyOn(navigation, "useRouter").mockReturnValue({ push: jest.fn() } as any);
```

## 与表单交互

使用 `@testing-library/user-event` 进行真实感表单交互（按正确顺序触发键盘、指针和焦点事件）：

```tsx
import userEvent from "@testing-library/user-event";

test("提交登录表单", async () => {
  const user = userEvent.setup();
  render(<LoginStep1 />);

  await user.type(screen.getByLabelText("用户名"), "B21041234");
  await user.click(screen.getByRole("button", { name: "下一步" }));

  expect(await screen.findByText("请输入密码")).toBeInTheDocument();
});
```

## 测试 Zustand Store

Zustand Store 可以直接导入并调用，无需渲染组件：

```ts
import { useAuthStore } from "@/store/use-auth-store";

test("退出登录后清除用户信息", () => {
  const store = useAuthStore.getState();
  store.setCurrentUser({ username: "Alice", email: "alice@example.com" });
  store.logout();
  expect(useAuthStore.getState().currentUser.username).toBe("");
});
```

在测试间重置 Store 状态，防止状态泄漏：

```ts
afterEach(() => {
  useAuthStore.setState({ currentUser: { username: "", email: "" }, loginTicket: null, redirect: null });
});
```

## 测试 API 函数

API 函数可以直接调用，MSW 会拦截 Axios 请求：

```ts
import { verifyLoginAccount } from "@/lib/api/auth";

test("已有用户返回 loginTicket", async () => {
  const res = await verifyLoginAccount("B21041234");
  expect(res.data.Success).toBe(true);
  if (res.data.Success) {
    expect(res.data.Data.loginTicket).toBeDefined();
  }
});
```

由于 `jest.setup.ts` 在所有测试前启动了 MSW 服务端，API 层测试无需额外配置即可拦截请求。
