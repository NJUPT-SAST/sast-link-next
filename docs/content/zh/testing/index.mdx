## 技术栈

| 工具 | 版本 | 用途 |
|---|---|---|
| Jest | 30 | 测试运行器与断言库 |
| jest-fixed-jsdom | 最新 | 修复了 async/structuredClone 问题的 DOM 环境 |
| ts-jest | 最新 | 测试文件的 TypeScript 转换 |
| @testing-library/react | 最新 | React 组件渲染与交互 |
| @testing-library/user-event | 最新 | 模拟用户交互 |
| MSW | 2 | 网络层 API 模拟 |

## 配置

### jest.config.ts

```ts
// 关键配置
testEnvironment: "jest-fixed-jsdom"
transform: { "^.+\\.(t|j)sx?$": ["ts-jest", { tsconfig: "tsconfig.json" }] }
moduleNameMapper: { "^@/(.*)$": "<rootDir>/$1" }  // 解析 @/* 导入
```

使用 `jest-fixed-jsdom` 而非标准 `jsdom`，因为它包含了 React 19 和部分测试模式所需的 `structuredClone` 等全局对象的补丁。

### jest.setup.ts

该配置文件在每个测试套件前执行，负责：

1. 导入 `@testing-library/jest-dom`，添加自定义匹配器（`toBeInTheDocument`、`toHaveClass` 等）。
2. 用简单 `<img>` 元素模拟 `next/image`。
3. 模拟 `next/navigation`（`useRouter`、`usePathname`、`useSearchParams`）。
4. 在所有测试前启动 MSW 服务端，测试间重置处理器，所有测试后关闭服务端。

### MSW 服务端生命周期

```ts
// jest.setup.ts
beforeAll(() => server.listen())
afterEach(() => server.resetHandlers())
afterAll(() => server.close())
```

`server.resetHandlers()` 会移除通过 `server.use(...)` 添加的一次性处理器覆盖，防止处理器状态在测试文件间泄漏。

## 覆盖率阈值

`pnpm test:coverage` 若未达到任一阈值则构建失败：

| 指标 | 阈值 |
|---|---|
| 分支覆盖率 | 60% |
| 函数覆盖率 | 60% |
| 行覆盖率 | 70% |
| 语句覆盖率 | 70% |

覆盖率从 `app/`、`components/`、`lib/`、`store/` 和 `hooks/` 下所有 `*.ts` 和 `*.tsx` 文件收集，排除测试文件、类型定义文件和生成代码。

## 测试文件共置规范

测试文件与源文件共置，使用 `.test.ts` 或 `.test.tsx` 扩展名：

```
components/auth/auth-shell.tsx
components/auth/auth-shell.test.tsx

lib/api/auth.ts
lib/api/auth.test.ts

store/use-auth-store.ts
store/use-auth-store.test.ts
```

页面的高层集成测试与页面文件共置：

```
app/(tourist)/login/page.tsx
app/(tourist)/login/page.test.tsx
```

## 执行测试

```bash
# 执行一次全部测试
pnpm test

# 监视模式：文件保存后仅重新执行受影响的测试
pnpm test:watch

# 运行并生成覆盖率报告
pnpm test:coverage
```
