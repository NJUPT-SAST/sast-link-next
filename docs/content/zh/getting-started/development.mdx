---
title: 开发命令
description: 全部可用的 pnpm 脚本及其说明
---

除非特别说明，所有命令均在仓库根目录执行。

## Web 应用

```bash
# 在 3000 端口启动支持 Fast Refresh 的 Next.js 开发服务器
pnpm dev

# 将应用构建为静态产物，输出至 out/ 目录
# pnpm tauri build 执行前必须先运行此命令；CI 中同样会用到
pnpm build

# 启动已构建的静态产物（需先执行 pnpm build）
pnpm start
```

## 代码质量

```bash
# 对整个代码库执行 ESLint 检查（配置文件：eslint.config.mjs）
pnpm lint

# 仅进行 TypeScript 类型检查，不生成产物
pnpm exec tsc --noEmit
```

## 测试

```bash
# 执行一次全部 Jest 测试
pnpm test

# 以监视模式运行 Jest，文件保存后仅重新执行受影响的测试
pnpm test:watch

# 运行 Jest 并收集覆盖率，未达到以下阈值时构建失败：
#   branches: 60%  functions: 60%  lines: 70%  statements: 70%
pnpm test:coverage
```

## Tauri 桌面端

```bash
# 以开发模式启动 Tauri 桌面应用。
# 内部先执行 pnpm dev（beforeDevCommand），再打开 WebView。
pnpm tauri dev

# 为当前平台构建 Tauri 桌面安装包。
# 内部先执行 pnpm build（beforeBuildCommand）生成 out/ 目录。
pnpm tauri build

# 打印 Tauri 环境信息（Rust 工具链、操作系统、WebView 版本等）
pnpm tauri info
```

## shadcn/ui 组件

```bash
# 向 components/ui/ 添加新的 shadcn/ui 风格基础组件
pnpm dlx shadcn@latest add <component-name>
```

## 文档站点

文档站点位于 `docs/` 子目录，拥有独立的 `package.json`。

```bash
cd docs

# 启动文档开发服务器（默认端口 3000）
pnpm dev

# 构建文档站点生产版本
pnpm build
```

## 构建产物

主应用的 `pnpm build` 会在 `out/` 目录生成完整的静态站点。Tauri 在生产环境通过 `src-tauri/tauri.conf.json` 加载该目录：

```json
{
  "build": {
    "frontendDist": "../out"
  }
}
```
