## 分支命名

所有分支须使用以下前缀之一：

| 前缀 | 适用场景 |
|---|---|
| `feat/` | 新功能 |
| `fix/` | 缺陷修复 |
| `docs/` | 仅文档修改 |
| `test/` | 添加或更新测试 |
| `refactor/` | 不涉及行为变更的代码重构 |
| `ci/` | CI/CD 流水线修改 |
| `chore/` | 依赖更新、工具链、配置变更 |

示例：`feat/oauth-qq-integration`、`fix/register-step2-resend-cooldown`、`docs/state-management`。

## 提交信息

提交遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

```
<type>(<可选 scope>): <简短描述>

<可选正文>

<可选页脚>
```

**type** 须为以下之一：`feat`、`fix`、`docs`、`test`、`refactor`、`ci`、`chore`、`style`、`perf`。

**scope** 可选，标识受影响的模块或功能（如 `auth`、`profile`、`store`、`api`）。

**描述** 使用祈使句，小写字母开头，末尾不加句号。

示例：
```
feat(auth): 添加飞书 OAuth 回调处理
fix(store): 修正 removeAccount 过滤器返回值
test(api): 补充 verifyResetAccount 覆盖率
docs: 补充密码重置流程文档
chore: 升级 Zustand 至 5.0.12
```

## Pull Request 要求

PR 合并前须通过以下三项检查：

1. **Lint**：`pnpm lint` 必须以代码 0 退出，不允许 ESLint 错误（警告可接受）。
2. **测试**：`pnpm test:coverage` 须通过，包括所有覆盖率阈值（分支 60%，函数 60%，行 70%，语句 70%）。
3. **构建**：`pnpm build` 须无错误完成，生成有效的 `out/` 目录。

这些检查由 `quality.yml` 和 `test.yml` GitHub Actions 工作流在每个 PR 上强制执行。

## 添加组件

添加新的 shadcn/ui 风格基础组件时：

```bash
pnpm dlx shadcn@latest add <component-name>
```

会在 `components/ui/` 生成一个文件，提交前需检查并调整以符合项目的 Tailwind v4 CSS 变量规范。

## 添加新路由

1. 在对应路由组（`(tourist)/` 或 `(user)/`）下创建文件夹。
2. 添加 `page.tsx`；若页面有私有子组件，放置于 `_components/` 子目录。
3. 添加对应的 `page.test.tsx`。
4. 若路由需要新布局，添加 `layout.tsx`。
5. 若路由引入了新的用户功能，更新本文档。

## 环境变量

绝不提交 `.env.local` 或任何含真实凭据的文件，仓库 `.gitignore` 已排除 `.env*.local`。使用 `.env.example`（如存在）记录所需变量（不含实际值）。
