## 概述

所有 CI/CD 均运行在 GitHub Actions 上，工作流定义于 `.github/workflows/`。

## quality.yml

由 `ci.yml` 调用，也支持手动触发。

步骤：
1. 检出代码。
2. 安装 pnpm 和 Node.js。
3. 执行 `pnpm install`。
4. 执行 `pnpm lint`。
5. 执行 `pnpm exec tsc --noEmit`。

## test.yml

由 `ci.yml` 调用，也支持手动触发。

步骤：
1. 检出代码。
2. 安装 pnpm 和 Node.js。
3. 执行 `pnpm install`。
4. 执行 `pnpm test:coverage`。
5. 执行 `pnpm build`。
6. 上传覆盖率与 `out/` 构建产物。

## deploy.yml（可选）

用于静态托管部署的可选工作流，默认禁用。启用前需配置目标环境与仓库 Secrets。

## release.yml

在匹配 `v*` 的标签（如 `v1.0.0`）上触发。

步骤：
1. 执行 `quality.yml`。
2. 执行 `test.yml`。
3. 下载工作流产物。
4. 创建 draft GitHub Release，并附加构建产物。

## 所需 Secrets

| Secret | 使用方 | 说明 |
|---|---|---|
| `GITHUB_TOKEN` | 所有工作流 | GitHub Actions 自动提供 |
| 部署凭据 | `deploy.yml` | 仅启用部署时按目标环境配置 |
