## 概述

客户端状态由 [Zustand 5](https://zustand.docs.pmnd.rs/) 管理，共四个 Store，各有明确职责：

| Store | 文件 | 是否持久化 | 用途 |
|---|---|---|---|
| `useAuthStore` | `store/use-auth-store.ts` | 是（部分） | 认证身份与登录流程状态 |
| `useUserListStore` | `store/use-user-list-store.ts` | 是（全部） | 账号切换器的已记住账号列表 |
| `useUserProfileStore` | `store/use-user-profile-store.ts` | 否 | 已认证页面使用的完整资料数据 |
| `usePanelStore` | `store/use-panel-store.ts` | 否 | 首页侧边面板的 UI 状态 |

## useAuthStore

追踪当前已识别用户与临时登录流程状态。

```ts
interface AuthState {
  currentUser: CurrentUserProfile;  // { username, email, userId?, avatar?, ... }
  loginTicket: string | null;       // verifyLoginAccount 返回的票据；登录后清除
  redirect: string | null;          // 登录成功后的跳转 URL

  setCurrentUser(user: CurrentUserProfile): void;
  clearCurrentUser(): void;
  setLoginTicket(ticket: string): void;
  setRedirect(redirect: string): void;
  clearLoginState(): void;          // 清除 loginTicket 和 redirect
  logout(): void;                   // clearToken() + 重置所有字段
}
```

**持久化**：`persist` 中间件仅将 `currentUser` 写入 localStorage，键名为 `"auth-store"`。`loginTicket` 和 `redirect` 仅在内存中，页面刷新后不会恢复。

**退出行为**：`logout()` 调用 `clearToken()`（从 localStorage 移除会话令牌），在单次 `set` 调用中将 `currentUser`、`loginTicket` 和 `redirect` 重置为初始值。

## useUserListStore

维护本设备上已登录过的账号列表，支持根页面的账号切换功能。

```ts
interface UserListState {
  accounts: UserAccount[];          // { token, avatar, nickName, email, userId }

  addAccount(account: UserAccount): void;     // userId 已存在则忽略
  updateAccount(update: { email: string; nickName: string; avatar: string | null }): void;
  removeAccount(identifier: number | string): void; // 按数组索引或邮箱删除
}
```

**持久化**：整个 `accounts` 数组持久化至 localStorage，键名为 `"user-list-store"`。

**去重**：`addAccount` 在追加前检查 `userId`，若已存在相同 `userId` 的账号则直接返回。使用 `updateAccount` 更新已有条目的显示名称和头像。

**删除**：`removeAccount` 接受数组索引（数字）或邮箱（字符串）。当不知道确切位置时使用基于邮箱的形式。

## useUserProfileStore

存储 `GET /profile/getProfile` 返回的完整 `UserProfileType`，供所有已认证页面读取资料，避免额外 API 调用。

```ts
interface UserProfileState {
  profile: UserProfileType;

  setProfile(profile: UserProfileType): void;      // 替换整个 profile
  updateProfile(fields: EditableFields): void;     // 合并部分字段
  resetProfile(): void;                            // 还原至初始占位值
}
```

**初始值**：所有字符串字段设为 `"~"`，可空字段设为 `null`，提供可识别的占位状态。

**不持久化**：资料数据始终在挂载时通过 API 重新拉取（via `useFetchProfile`），持久化会有返回陈旧数据的风险。

**`updateProfile`**：使用对象展开将提供的 `EditableFields` 合并至当前 profile，在编辑页 `editProfile` API 调用成功后使用，避免完整重新拉取。

## usePanelStore

控制首页信息侧边面板的简单 UI 状态 Store。

```ts
interface PanelState {
  homeInfoPanel: boolean;
  setHomeInfoPanel(open: boolean): void;
}
```

不持久化。每次页面加载时面板默认关闭（`false`）。

## 数据流示例

以下展示登录成功后 Store 的填充过程：

```
1. userLogin() 返回 loginToken
        |
2. setToken(loginToken)               → localStorage
3. useAuthStore.clearLoginState()
        |
4. 导航至 /home
        |
5. useFetchProfile()（在首页布局中调用）
        |
6. GET /profile/getProfile
        |
7. useUserProfileStore.setProfile(data)
8. useAuthStore.setCurrentUser({ username: data.nickname, email: data.email })
9. useUserListStore.updateAccount({ email, nickName, avatar })
```
