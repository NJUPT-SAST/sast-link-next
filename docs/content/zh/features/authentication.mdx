title: 认证
description: 基于票据的两步登录流程
---

## 概述

登录分为两步。第一步验证账号是否存在并签发短期票据，第二步使用票据与密码换取会话令牌。

这种设计意味着密码不在第一步请求中发送，后端可以对每个票据独立施加频率限制和暴力破解防护。

## 第一步：账号验证

用户输入用户名或邮箱。提交后，前端调用 `verifyLoginAccount(username)`，请求 `GET /verify/account?username=...&flag=1`。

成功响应返回 `loginTicket` 字符串。票据通过 `setLoginTicket(ticket)` 存入 `useAuthStore`，流程进入第二步。

```
用户输入用户名/邮箱
        |
verifyLoginAccount(username)
        |
GET /verify/account?flag=1
        |
{ Success: true, Data: { loginTicket: "..." } }
        |
useAuthStore.setLoginTicket(ticket)
        |
进入第二步
```

若账号不存在或后端返回错误，则显示内联错误提示，流程停留在第一步。

### OAuth 快速登录

第一步还渲染 `OtherLoginList` 组件，展示 OAuth 服务商按钮（GitHub、飞书、QQ、微软）。点击服务商按钮后，浏览器跳转至对应的授权页面，授权完成后服务商重定向回 `/callback/feishu` 或 `/callback/github`。

## 第二步：密码与令牌交换

用户输入密码。提交后，前端调用 `userLogin(password, loginTicket, oauthTicket?)`，向 `/user/login` 发送 POST 请求，包含：

- 请求体中的 `password`。
- `LOGIN-TICKET` 请求头，设置为第一步获取的票据。
- `OAUTH-TICKET` 请求头（可选），仅在 OAuth 绑定流程时存在。

成功响应返回 `loginToken`，通过 `lib/token.ts` 的 `setToken(token)` 持久化至 localStorage。

```
用户输入密码
        |
userLogin(password, loginTicket)
        |
POST /user/login
  body: password
  headers: LOGIN-TICKET, (OAUTH-TICKET 如有)
        |
{ Success: true, Data: { loginToken: "..." } }
        |
setToken(loginToken)            ← localStorage
useAuthStore.clearLoginState()  ← 清除票据与重定向
        |
跳转至 /home（或已存储的重定向 URL）
```

## 与认证 Store 的交互

`useAuthStore`（通过 `persist` 中间件持久化至 localStorage 的 `"auth-store"` 键）追踪登录流程状态：

| 字段 | 设置方 | 清除方 |
|---|---|---|
| `loginTicket` | 第一步后调用 `setLoginTicket` | 第二步后调用 `clearLoginState`，或 `logout` |
| `redirect` | 导航至 `/login` 前调用 `setRedirect` | 第二步后调用 `clearLoginState`，或 `logout` |
| `currentUser` | 资料拉取后调用 `setCurrentUser` | `clearCurrentUser`，或 `logout` |

只有 `currentUser` 被 `persist` 中间件写入 localStorage。票据和重定向仅在内存中，页面刷新后丢失——这是有意为之，旧会话中的陈旧票据不应被复用。

## 校验

表单输入在发起 API 调用前通过 Zod 进行校验：

```ts
// lib/validations/auth.ts
export const loginAccountSchema = z.union([
  z.string().regex(/^\d{9}$/, "须为 9 位学号"),
  z.string().email("须为有效邮箱地址"),
]);

export const loginPasswordSchema = z.string().min(1, "密码不能为空");
```

## 退出登录

调用 `useAuthStore.logout()` 时：
1. 调用 `clearToken()` 从 localStorage 移除令牌。
2. 将 `currentUser` 重置为 `{ username: "", email: "" }`。
3. 清除 `loginTicket` 和 `redirect`。

调用组件负责在退出后导航至 `/login`。`userLogout()` API 调用可在 Store 重置前后进行。
