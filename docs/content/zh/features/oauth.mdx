## 支持的服务商

sast-link-next 集成两个 OAuth 服务商用于第三方登录：

| 服务商 | 回调路由 | 后端端点 |
|---|---|---|
| GitHub | `/callback/github` | `GET /login/github/callback` |
| 飞书（Lark） | `/callback/feishu` | `GET /login/lark/callback` |

`components/auth/other-login-list.tsx` 中的 `OtherLoginList` 组件渲染 GitHub、飞书、QQ 和微软的登录按钮。QQ 和微软已展示但当前版本尚未实现其流程。

## OAuth 授权流程（PKCE）

当应用本身作为 OAuth 客户端（即第三方应用请求通过 SAST Link 进行授权）时，使用 `lib/api/oauth.ts` 中的 `oAuth()` 函数构建授权 URL 并跳转浏览器。

```ts
// lib/api/oauth.ts
export function oAuth(data: OAuthParams) {
  const params = new URLSearchParams();
  for (const [key, value] of Object.entries(data)) {
    if (value) params.append(key, value);
  }

  const token = getToken();
  if (token) {
    params.append("part", token);
  }

  window.location.href = `${API_BASE_URL}/oauth2/authorize?${params.toString()}`;
}
```

`OAuthParams` 接口接受标准 PKCE 参数：

| 参数 | 说明 |
|---|---|
| `client_id` | OAuth 应用标识符 |
| `code_challenge` | PKCE code challenge（S256） |
| `code_challenge_method` | 固定为 `"S256"` |
| `redirect_uri` | 授权服务器审批后的重定向地址 |
| `response_type` | 固定为 `"code"`（授权码流程） |
| `scope` | 申请的权限范围 |
| `state` | 用于 CSRF 防护的不透明值 |

若会话令牌存在，则将其作为 `part` 参数附加，使授权服务器能够识别当前登录用户，无需额外登录步骤。

## 社交登录回调流程

用户在登录页点击 GitHub 或飞书登录后，浏览器跳转至服务商授权端点。用户授权完成后，服务商携带 `code` 和 `state` 参数重定向回对应的回调路由。

回调页面调用相应的后端端点：

```ts
// GitHub
getGithubLoginStatus(code, state)
// → GET /login/github/callback?code=...&state=...

// 飞书
getFeishuLoginStatus(code, state)
// → GET /login/lark/callback?code=...&state=...
```

后端将 code 换取服务商凭证，返回以下两种响应之一：

1. **直接登录**：`{ loginToken: "..." }` — 用户已有关联此服务商的 SAST Link 账号，令牌存储后跳转至 `/home`。
2. **账号绑定**：`{ oauthTicket: "..." }` — 该服务商账号尚未关联，OAuth 票据存储后跳转至登录流程第二步（输入密码）。用户输入密码后，`userLogin` 携带 `loginTicket` 和 `oauthTicket` 一并调用，后端据此将服务商账号与 SAST Link 账号关联。

## 绑定状态

`getUserBindStatus()` 调用 `GET /profile/bindStatus`，返回已关联至当前用户账号的服务商标识字符串数组。通过 `BindAppItem` 组件在资料页面展示。
