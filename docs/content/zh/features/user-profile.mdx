## 首页（/home）

首页展示已登录用户的资料摘要，并提供导航至编辑和设置页面的快捷入口。

### 布局结构

`app/(user)/home/layout.tsx` 是顶层已认证布局，负责：
1. 挂载时调用 `useFetchProfile()` 将资料数据加载至 Zustand Store。
2. 在顶部渲染 `TopBar`。
3. 通过并行路由将主内容、`@infoPanel` 和 `@profilePanel` 排列在响应式网格中。

### 页面内容

`app/(user)/home/page.tsx` 渲染：

- **资料头部**：来自 `useUserProfileStore` 的头像、昵称、部门与组织信息。
- **资料完整度**：可选字段填写情况的可视化指示器。
- **快捷操作**：跳转至 `/home/edit`（编辑资料）和 `/home/edit/safety`（安全设置）的按钮。
- **简介与链接**：用户的简介文本及最多三个自定义链接。
- **常用应用列表**：固定链接至四个集成应用——审批、OJ（在线评测）、视觉艺术与 Evento。

## 资料编辑（/home/edit）

`app/(user)/home/edit/page.tsx` 提供两个功能区块：

### 头像上传与裁剪

头像编辑器使用 `react-avatar-editor`，让用户在上传前对图片进行裁剪和缩放。

- 用户通过文件输入选择图片。
- 编辑器渲染圆形裁剪画布。
- **缩放**：滚轮调整缩放比例，范围为 1 倍（最小，不缩放）至 5 倍（最大）。
- 用户点击"保存"，将裁剪后的图片导出为 `Blob` 并通过 `uploadAvatar(blob)` 上传。
- `uploadAvatar` 以 `multipart/form-data` 格式向 `POST /profile/uploadAvatar` 发送请求，字段名为 `avatarFile`。
- 成功后，响应中包含 `filePath`（新头像 URL），随后更新至资料 Store。

### 资料字段表单

表单使用 React Hook Form 与 Zod 校验（`lib/validations/profile.ts`）：

| 字段 | 约束 |
|---|---|
| 昵称 | 最多 20 个字符 |
| 简介 | 最多 200 个字符 |
| 链接 1 | 有效 URL 或留空 |
| 链接 2 | 有效 URL 或留空 |
| 链接 3 | 有效 URL 或留空 |

提交后，`editProfile(data)` 向 `POST /profile/changeProfile` 发送更新字段。成功响应后，调用 `useUserProfileStore.updateProfile(fields)` 保持 Store 同步，无需完整重新拉取。

发送至后端的 `EditableProfileType`：

```ts
interface EditableProfileType {
  nickname?: string | null;
  org_id?: number;
  bio?: string;
  link?: string[] | null;
  hide?: string[] | null;
}
```

## 安全设置（/home/edit/safety）

`app/(user)/home/edit/safety/page.tsx` 当前展示：

- **邮箱**：用户已注册的邮箱地址（只读），来源于 `useUserProfileStore`。
- **邮箱换绑**：占位区块，实际换绑流程在当前版本尚未实现。

## 资料页面数据流

```
useFetchProfile()（在 /home 布局中调用）
        |
GET /profile/getProfile
        |
useUserProfileStore.setProfile(data)
useAuthStore.setCurrentUser({ username, email })
useUserListStore.updateAccount({ email, nickName, avatar })
        |
所有资料页面从 useUserProfileStore 同步读取
```

由于资料在布局层面统一拉取并写入 Store，所有嵌套页面（`/home`、`/home/edit`、`/home/edit/safety`）均可从 `useUserProfileStore` 同步读取资料，无需额外 API 调用。
