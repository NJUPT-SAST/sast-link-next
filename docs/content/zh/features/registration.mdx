## 概述

新账号通过四步注册流程创建。每步仅在当前步骤的校验与 API 调用成功后才会推进。步骤通过 `app/(tourist)/register/page.tsx` 中的本地 `step` 状态追踪。

```
第一步：输入学号  →  第二步：验证邮箱  →  第三步：设置密码  →  第四步：注册成功
```

## 第一步：输入学号

用户输入 9 位学号。提交后，前端：

1. 推导机构邮箱地址：`studentId@njupt.edu.cn`（通过 `lib/validations/auth.ts` 中的 `studentIdToEmail(studentId)`）。
2. 调用 `verifyRegistAccount(username)`，请求 `GET /verify/account?username=...&flag=0`。
3. 成功后获取 `registerTicket`。
4. 调用 `sendMail(registerTicket)` 发送验证邮件。
5. 存储票据并进入第二步。

```ts
// lib/validations/auth.ts
export const registerStudentIdSchema = z.string().regex(/^\d{9}$/, "须为 9 位数字");

export function studentIdToEmail(studentId: string): string {
  return `${studentId}@njupt.edu.cn`;
}
```

推导出的邮箱地址会展示给用户，告知其验证码将发送至何处。

## 第二步：邮箱验证

用户输入收到的 5 位验证码。前端在发送前自动加上固定前缀 `"S-"`：

```ts
// lib/api/auth.ts
params.append("captcha", "S-" + captcha);
```

请求为 `POST /verify/captcha`，携带 `REGISTER-TICKET` 请求头与 `application/x-www-form-urlencoded` 请求体。

重发按钮有 60 秒冷却时间，由 `VerificationCodeInput` 组件内的倒计时器控制。计时器归零后按钮重新启用，再次调用 `sendMail(ticket)`。

`verificationCodeSchema` 会在提交前校验输入是否恰好为 5 位数字。

## 第三步：设置密码

用户设置密码。校验规则：

- 最少 8 个字符。
- 必须包含至少一个字母和一个数字。
- 确认密码字段必须与密码完全一致。

```ts
// lib/validations/auth.ts
export const passwordSchema = z
  .string()
  .min(8, "至少 8 个字符")
  .regex(/[a-zA-Z]/, "须包含字母")
  .regex(/[0-9]/, "须包含数字");

export const registerPasswordSchema = z
  .object({ password: passwordSchema, confirmPassword: z.string() })
  .refine((d) => d.password === d.confirmPassword, {
    message: "两次密码不一致",
    path: ["confirmPassword"],
  });
```

提交后，`userRegister(password, registerTicket)` 向 `POST /user/register` 发起请求，请求体包含密码，请求头包含 `REGISTER-TICKET`。成功后进入第四步。

## 第四步：注册成功

第四步是一个带有动画对勾的确认页面，包含指向 `/login` 的链接，用户可立即登录新账号。此步骤不发起任何 API 调用。

## 错误处理

每步均通过以下来源展示内联错误信息：
- Zod 校验（通过 React Hook Form 显示字段级错误）。
- API 错误响应（后端返回的 `ResType.ErrMsg`）。

若 API 调用失败，步骤不会推进，用户可重试。
